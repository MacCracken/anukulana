# anukulana ‚Äî v1.0 Security / Hardening Audit (2026-07-04)

> Pre-1.0 audit of the untrusted-input and trust-boundary surfaces
> (`docs/api.md`): `src/safetensors.cyr` line-by-line (the foreign parser ‚Äî
> THE attack surface), `src/oracle.cyr` fixture reader, `src/nf4_store.cyr`
> load paths, plus the numeric modules' precondition story. Method: manual
> derive-from-source pass + a new deterministic fuzz gate
> (`tests/tcyr/fuzz.tcyr`, 38k+ adversarial rounds). Two findings, **both
> fixed in this pass**.

## Verdict

**PASS after 2 fixes** ‚Äî both in `st_open` (the foreign-safetensors parser),
both hostile-input DoS class, neither reachable from well-formed files:

1. **Integer-wrap in the header bounds check.** `if ((8 + hlen) > len)`
   overflows for `hlen` within 8 of i64-max ‚Äî those 8 pathological values
   passed the check and reached the JSON parser with a wild length. Fixed to
   the wrap-safe form `hlen > len - 8` (both operands proven in range by the
   preceding checks).
2. **Unchecked allocation for the entry table.** `alloc(nobj * 112)` with a
   hostile header declaring millions of members exceeds the 256 MB allocator
   cap, `alloc` returns 0, and the subsequent entry stores write near address
   0 ‚Üí crash (DoS, not corruption ‚Äî the writes land in unmapped low memory).
   Fixed with an explicit `entries == 0 ‚Üí reject`.

## What was verified clean

- **`st_open` (post-fix):** header length double-checked; JSON via bayan (no
  hand-rolled scanning); per-tensor `data_offsets` validated `0 ‚â§ o0 ‚â§ o1 ‚â§
  data_len` (no wrap path ‚Äî `data_len < len`, offsets checked before use);
  `ndim ‚â§ 8`; dtype whitelist; `__metadata__` skipped; every reject is a clean
  `return 0`. Accessors only expose entries that passed validation, and
  `st_entry_data` arithmetic is bounded by the validated offsets.
- **Fuzz gate (new, a v1.0 criterion):** 30k byte-mutation rounds over a valid
  image + a full truncation sweep + 5k garbage buffers, with accessor probes
  on every accepted reader; plus 3k mutated/truncated oracle-fixture files.
  Deterministic LCG ‚Äî replayable. Zero crashes; rejections observed (the
  parser demonstrably validates rather than accepting everything).
- **`_orc_read_fixture`:** magic, seq-count cap, per-seq T/V bounds, uniform-T
  enforcement, payload-overrun checks ‚Äî all present; fuzz-covered.
- **`anuk_nf4_load` / `anuk_adapter_load`:** structural validation delegated to
  `tula_open` (tula's own frozen, fuzzed surface); **Ed25519 verification
  mandatory** ‚Äî unsigned/wrong-key/tampered rejected (gated in
  `nf4_store.tcyr`); shape/count cross-checks (`np` vs cfg, block count) before
  any dequantization; big buffers via `mmap_anon` with null-checks.
- **Numeric modules (`lora.cyr` / `nf4.cyr`):** operate on caller-sized buffers
  under rosnet's audited caller-guarantees contract ‚Äî no external input enters
  them except through the validated readers above; the NF4 base codec is
  tula's (its audit surface).
- **Large files:** mmap'd read-only past the alloc cap; no unbounded heap read
  of attacker-sized input (`st_read_file` takes an explicit `maxlen`;
  `tula_read_file` bomb-guards).

## Out of scope

sigil's crypto internals; tula's format internals (own audit); rosnet's
substrate contract (own audit, 2026-07-04); adversarial *weights* semantics
(serving-layer policy ‚Äî hoosh/kavach); key management (caller's, until the
control plane lands it).

## Standing guidance for 1.x

Any new foreign-input parser (**GGUF is the named post-1.0 addition**) must:
validate every length/offset wrap-safely BEFORE use, null-check every
allocation sized from attacker data, reject with `return 0` (never partial
readers), and land with its own fuzz rounds in `tests/tcyr/fuzz.tcyr` in the
same cut.
